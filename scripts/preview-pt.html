📋 Resumo

Este PR adiciona lógica de retry com backoff exponencial ao PaymentService, corrigindo falhas silenciosas de pagamento causadas por timeouts no gateway. A implementação é bem estruturada, com boa separação de responsabilidades e testes cobrindo os cenários principais. A única preocupação bloqueante é a detecção de erros não-retentáveis por correspondência de string, que pode causar cobranças duplicadas em produção.

✅ Pontos Positivos
  • Ótima separação: withRetry é um utilitário genérico e reutilizável, enquanto isNonRetryable encapsula a lógica de domínio de pagamentos
  • RetryExhaustedError preserva o lastError original, facilitando muito o diagnóstico em produção
  • Testes cobrem os 4 cenários críticos: sucesso, retry em falha transiente, esgotamento e bypass de retry
  • Uso correto de Math.pow(2, attempt - 1) para backoff — evita sobrecarga no gateway em falhas em massa
🚨 Problemas Críticos
  • src/services/payment.service.ts:18 — isNonRetryable usa err.message.includes(code) para detectar erros não-retentáveis. Se o gateway de pagamento alterar o texto da mensagem de erro, cartões recusados serão retentados 3 vezes, podendo causar cobranças duplicadas em produção. Use códigos de erro estruturados.
⚠️ Melhorias Sugeridas
  • Adicionar jitter ao delay — delay * (0.8 + Math.random() * 0.4) — para evitar thundering herd quando múltiplos pagamentos falham ao mesmo tempo
  • RetryExhaustedError nunca é lançado — o loop relança lastError diretamente. Remova a classe ou corrija o fluxo para usá-la
  • Logar tentativas de retry em nível warn para observabilidade em produção
  • Expor maxDelayMs como opção para limitar o delay em cenários de alta latência
💡 Sugestões com Código

Use códigos estruturados em vez de correspondência de string:

const NON_RETRYABLE_CODES = new Set([
  'card_declined',
  'invalid_cvv',
  'expired_card',
  'insufficient_funds',
]);

function isNonRetryable(err: unknown): boolean {
  if (err instanceof GatewayError) {
    return NON_RETRYABLE_CODES.has(err.code); // seguro, o código não muda silenciosamente
  }
  return false;
}

Adicionar jitter para evitar thundering herd:

// Antes: const delay = baseDelayMs * Math.pow(2, attempt - 1);
const jitter = 0.8 + Math.random() * 0.4; // aleatório entre 80%–120%
const delay = baseDelayMs * Math.pow(2, attempt - 1) * jitter;
