📋 Summary

This PR adds exponential backoff retry logic to PaymentService, fixing silent payment failures caused by gateway timeouts. The implementation is well-structured with a clean separation of concerns and tests covering the key scenarios. The only blocking concern is non-retryable error detection via string matching, which could cause duplicate charges in production.

✅ What's Good
  • Clean separation: withRetry is a generic, reusable utility while isNonRetryable encapsulates payment domain logic
  • RetryExhaustedError preserves the original lastError, making production debugging much easier
  • Tests cover all 4 critical scenarios: success, retry on transient failure, exhaustion, and retry bypass
  • Correct use of Math.pow(2, attempt - 1) for backoff — avoids gateway overload on burst failures
🚨 Critical Issues
  • src/services/payment.service.ts:18 — isNonRetryable uses err.message.includes(code) to detect non-retryable errors. If the payment gateway changes its error message wording, declined cards will be retried 3 times, potentially causing duplicate charges in production. Use structured error codes instead.
⚠️ Suggestions
  • Add jitter to the delay — delay * (0.8 + Math.random() * 0.4) — to avoid thundering herd when multiple payments fail at the same time
  • RetryExhaustedError is never thrown — the loop re-throws lastError directly. Either remove the class or fix the control flow to use it
  • Log retry attempts at warn level for production observability
  • Expose maxDelayMs as an option to cap the delay in high-latency scenarios
💡 Code Suggestions

Use structured error codes instead of string matching:

const NON_RETRYABLE_CODES = new Set([
  'card_declined',
  'invalid_cvv',
  'expired_card',
  'insufficient_funds',
]);

function isNonRetryable(err: unknown): boolean {
  if (err instanceof GatewayError) {
    return NON_RETRYABLE_CODES.has(err.code); // safe, code won't change silently
  }
  return false;
}

Add jitter to prevent thundering herd:

// Before: const delay = baseDelayMs * Math.pow(2, attempt - 1);
const jitter = 0.8 + Math.random() * 0.4; // random between 80%–120%
const delay = baseDelayMs * Math.pow(2, attempt - 1) * jitter;
